Wait a minute, I just used an unfamiliar word: repository. Essentially, a repository is the Git/GitHub way to say “directory” or “folder”. A repository is usually one project; one repository contains all of the files and information for one project. For example, the files for this book can all be found under my repository titled gitstarted 32 . This book constitutes as one project and therefore all files for this book are there. Remember Jennifer Bryan from above? All of the files for her book, Happy Git With R can be found on her repository 33 . Soon, you too will have repositories! (Note: a repository is often shortened to just repo.)
